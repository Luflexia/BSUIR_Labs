call - вызов процедуры (передача управления)

1. Директивы определения данных

string db 202 dup("$")
Идентификатор Тип Размер DUP (Значение)

Идентификатор — имя массива;
Тип — определяет количество байт, занимаемое одним элементом;
Размер — константа, характеризующая количество элементов в массиве
Значение — начальное значение элементов.


Директива EQU предназначена для описания констант


2. Команды пересылки данных и способы адресации.
Базовой командой пересылки данных является команда MOV:
MOV приемник, источник

-- PUSH источник – поместить данные в стек,
-- POP приемник – считать данные из стека

Операторы адресации:
-- SEG выражение – сегментный адрес;
-- OFFSET выражение – смещение;
-- THIS тип – текущий адрес (TASM и MASM);
-- тип PTR выражение – переопределение типа;
-- SMALL выражение – 16-ти битное смещение (TASM);
-- SHORT выражение – 8-ми битное смещение.


3. Команды передачи управления.

-- безусловный переход – переход на метку от текущего положения  (JMP)
-- условный переход - переход в зависимости от состояния флагов (Jcc)
-- переход, если CX = 0 (JCXZ)
-- цикл (LOOP) – организация циклов в программах, используя регистр CX 
в качестве счетчика цикла. Команда LOOP уменьшает содержимое CX на 1


4. Строковые операции.

-- B (byte)
-- W (word)
-- сравнение строк (CMPS)
-- сканирование строки (SCAS) – команда производит сравнение содержимого аккумулятора с байтом или словом памяти
-- пересылка строки (MOVS) – пересылает поэлементно строку DS:SI встроку ES:DI 
-- запись в строку (STOS) – заполняет строку, содержащуюся по адресуES:DI, элементом из аккумулятора
-- чтение из строки (LODS) – записывает в аккумулятор содержимое ячейки памяти

-- повторять, пока равно (REPE),
-- повторять, пока ноль (REPZ),
-- повторять (REP),
-- повторять, пока не равно (REPNE),
